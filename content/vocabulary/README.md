# content/vocabulary

Source-of-truth vocabulary enrichment data for the immersion rollout.
Files here back the importers in Tasks 35-42 of
`docs/plans/2026-05-11-post-immersion-content-data-plan.md` and only
those importers may write to the DB.

Companion documents:

- `docs/content/immersion-content-targets.md` Section 4 — coverage
  targets per priority tier.
- `docs/runbooks/immersion-data-rollback.md` — restore procedure for
  `collection_words`, `word_collocations`, `cultural_notes`, and
  `user_card_directions.source`.

## What belongs here

| File                       | Purpose                                                                       | Generated / Authored |
| -------------------------- | ----------------------------------------------------------------------------- | -------------------- |
| `priority_words.csv`       | Tiered priority list exported by `scripts/export_vocabulary_priority.py`.     | Generated            |
| `frequency_bands.csv`      | `word_id, band` rows. Allowed values: `1`, `2`, `3`.                          | Authored             |
| `ipa_transcriptions.csv`   | `word_id, ipa` rows. Stored without surrounding slashes.                      | Authored             |
| `synonyms_antonyms.csv`    | `word_id, synonyms_json, antonyms_json` rows. JSON list values only.          | Authored             |
| `collocations.csv`         | `word_id, collocation_phrase, translation, example`. Reused by collocation lessons. | Authored      |
| `etymology_notes.csv`      | `word_id, note` rows. Concise, no academic prose.                             | Authored             |
| `cultural_notes.csv`       | `word_id, context, note` rows for idioms / phrasal verbs / politeness terms.  | Authored             |

Generated files have a header comment block at the top of each CSV
identifying the generator script, generation timestamp, and the
priority tier filter applied. Do not hand-edit generated files;
regenerate them.

## What does NOT belong here

- Audio binaries or assets.
- Full corpus dumps from the 24,853-row `collection_words` table.
- Translations that belong to lesson content
  (those live in `content/immersion/`).
- User-generated vocabulary (`VocabAnnotation`, `user_card_directions`
  payloads). The SRS source backfill (Task 42) operates directly on
  `user_card_directions.source` using a script with no input file —
  it reads the DB rather than this directory.

## File format conventions

- All files are CSV with a single header row.
- UTF-8 encoded, no BOM.
- Line endings: LF.
- One row per `(word_id, ...)` tuple. Upsert key is documented per
  importer below.
- JSON columns are valid JSON arrays (`["fast", "quick"]`), never
  Python-repr strings.

### Stable ids

`word_id` is the primary stable id and **always** refers to
`collection_words.id` in the application database. Vocabulary content
never invents its own `external_key` because the row already has a
durable DB id maintained by the curriculum import pipeline.

For files that may carry additional rows per word (collocations,
cultural notes), the upsert key is documented explicitly:

| File                  | Upsert key                                                 |
| --------------------- | ---------------------------------------------------------- |
| `frequency_bands.csv` | `word_id`                                                  |
| `ipa_transcriptions.csv` | `word_id`                                               |
| `synonyms_antonyms.csv` | `word_id`                                                |
| `collocations.csv`    | `(word_id, collocation_phrase)` — case-insensitive compare |
| `etymology_notes.csv` | `word_id`                                                  |
| `cultural_notes.csv`  | `(word_id, context)` — `context` is the situational tag    |

## Force overwrite policy

Importers preserve existing non-null DB values by default. Use
`--force` only when a content audit explicitly asks to overwrite — and
record the reason in the import report under `reports/`.

## Workflow

1. Refresh the priority list:
   ```bash
   python scripts/export_vocabulary_priority.py
   ```
2. Edit the appropriate CSV. Keep diff size small for review.
3. Dry-run the importer:
   ```bash
   python scripts/import_<file>.py --dry-run content/vocabulary/<file>.csv
   ```
4. Apply, then re-run `--dry-run`. The second run must report
   `0 changes`.
5. Capture coverage delta in `reports/vocabulary_enrichment.md`
   (generated by `scripts/report_vocabulary_enrichment.py`).
