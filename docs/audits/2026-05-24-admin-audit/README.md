# Admin full audit (2026-05-24)

Living findings journal for `docs/plans/2026-05-24-admin-full-audit.md`. Each task in the plan appends a short section here with the findings it discovered and the fix it applied.

## Snapshot artifacts (Task 1)

- `url-map.md` — every admin route (method, URL, view, module, wrapper chain) generated by `scripts/_audit_dump_admin_routes.py`.
- `inventory.md` — LOC inventory of `app/admin/`, services, templates, tests + missing-coverage list.

## Decorator baseline (Task 1)

- `admin_required` already existed in `app/admin/utils/decorators.py` and is the only admin gate in use. Task 1 normalises its docstring and adds `admin_audit_required(action_name, target_type_callable=None, target_id_callable=None)` — a thin wrapper that combines `admin_required` with a call to `log_admin_action()` on a successful 2xx response so destructive endpoints adopted in Tasks 2–4 get audit logging for free.
- `tests/admin/test_admin_url_inventory.py` enumerates every admin-prefixed route and asserts that anonymous / non-admin access redirects to login or returns 401/403/404. A short, whitelisted set of endpoints (e.g. the GSC OAuth callback) is documented inline.

5. Several sub-blueprints have no dedicated unit-tests (see `inventory.md` § "Missing coverage").
