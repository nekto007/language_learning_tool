# Реестр: 100 багов и edge cases (аудит 2026-06-13)

Источник: план `docs/plans/2026-06-13-100-edge-cases.md`. Только документирование — правок кода нет.
Образец формата: `docs/audit/2026-05-23-audit-findings.md`.

## Методика

- Полный `pytest` + `pytest -m smoke` для фиксации красных тестов и сверки с baseline.
- Чтение кода + grep по edge-сигнатурам: `None`/`[]`/`0`/boolean precedence, naive vs aware datetime,
  race/idempotency, off-by-one, таймзоны, деление на ноль, флаги precedence, snapping в полночь.
- Каждая находка ссылается на реальный `file:line` и воспроизводима по коду (без ложных срабатываний).
- Sweep по доменным кластерам (секции B–K), уникальные id `E-001…E-100+`.

## Легенда severity

- **P0** — data-loss / security / сломанный пользовательский сценарий.
- **P1** — UX-баг, тех-долг с реальным риском.
- **P2** — полировка, dead code, refactor-кандидат.

## Колонки

## L. Топ-риски P0

Шесть находок, требующих приоритетного фикса (data-loss / security / сломанный сценарий):

1. **E-046 — data-loss (books).** `BookModuleProgress.mark_lesson_completed` мутирует
   `Column(JSON)` in-place без `MutableList/MutableDict` → завершение 2-го и последующих
   уроков модуля **не персистится**. Тихая потеря прогресса у каждого, кто проходит
   книжный курс длиннее одного урока. Фикс: `MutableList.as_mutable(JSON)` или
   переприсваивание нового объекта.

2. **E-080 — security (upload).** `check_forbidden_magic_bytes` — prefix-only по offset 0:
   PHP-короткий тег `<?=`, ведущая пустая строка или `<script` не в начале проходят гейт.
   `.csv` с `<?=system($_GET['c']);?>` принимается. Фикс: матчить `<?`/`<script`/shebang
   в любом месте sample; гейт — best-effort, не отдавать загруженное как исполняемое.

3. **E-031 — broken scenario (final test).** MC-вопрос final-test сравнивает `int(user_answer)`
   с не-коэрсенным `correct_answer`: контент с `"answer":"2"` (строка) → `2 == "2"` → всегда
   False → юзер не может сдать final test, `attempts_exhausted` после 3 фантомных провалов.
   Фикс: коэрсить `correct_answer` к int (как `process_quiz_submission`).

4. **E-033 — broken scenario / 500 (final test matching).** `correct_pairs = {p['left']:p['right']}`
   жёсткий доступ; пары в форме `{english,russian}` → `KeyError` → необработанный 500,
   попытка не записывается. Фикс: multi-shape extractor (как `_grade_matching_pairs`).

5. **E-035 — broken scenario / 500 (grammar match).** Тот же жёсткий `pair['left']/['right']`
   в `process_grammar_submission` → `KeyError`/500 на submit грамматики типа `match`.
   Фикс: `.get('left') or .get('english')` либо валидация формы пар.

6. **E-092 — broken scenario (content↔grader).** Урок `sentence_correction A1:01`:
   `options` — фрагменты, `correct_sentence` — полное предложение; single-item грейдер
   сравнивает опцию с полным эталоном → правильная опция даёт `passed=False` → урок
   непроходим через options-UI. Фикс: привести контент (`options` = полный эталон) +
   content-validator `correct_sentence ∈ {normalize(o)}`.
