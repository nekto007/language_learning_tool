# Реестр: 100 багов и edge cases (аудит 2026-06-13)

Источник: план `docs/plans/2026-06-13-100-edge-cases.md`. Только документирование — правок кода нет.
Образец формата: `docs/audit/2026-05-23-audit-findings.md`.

## Методика

- Полный `pytest` + `pytest -m smoke` для фиксации красных тестов и сверки с baseline.
- Чтение кода + grep по edge-сигнатурам: `None`/`[]`/`0`/boolean precedence, naive vs aware datetime,
  race/idempotency, off-by-one, таймзоны, деление на ноль, флаги precedence, snapping в полночь.
- Каждая находка ссылается на реальный `file:line` и воспроизводима по коду (без ложных срабатываний).
- Sweep по доменным кластерам (секции B–K), уникальные id `E-001…E-100+`.

## Легенда severity

- **P0** — data-loss / security / сломанный пользовательский сценарий.
- **P1** — UX-баг, тех-долг с реальным риском.
- **P2** — полировка, dead code, refactor-кандидат.

## Колонки

`id | layer | severity | file:line | описание | repro | предлагаемый фикс`
